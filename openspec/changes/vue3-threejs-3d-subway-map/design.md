## Context

基于 Vue3 + TresJS 创建一个3D地铁可视化项目。现有项目中已有杭州地铁的原始数据（位于 `src/json/city.json`），需要将其渲染为3D图形，并实现列车在轨道上移动的动画效果。

项目使用 Vite + Vue3 + JavaScript 作为基础框架，**TresJS** 作为3D渲染引擎，**@tresjs/cientos** 提供常用3D组件，**@tresjs/post-processing** 提供酷炫的后期处理效果。

## Goals / Non-Goals

**Goals:**
- 初始只渲染1号线和2号线的3D轨道
- 在每个站点显示3D圆柱体或球体标注
- 实现列车模型沿轨道平滑移动的动画
- 支持鼠标旋转、缩放、拖拽的3D视角控制
- 响应式布局，适配不同屏幕尺寸

**Non-Goals:**
- 不实现站点详情弹窗或信息面板（后续可扩展）
- 不实现多种列车车型号的区分
- 不实现与真实地铁时刻表同步的实时数据

## Decisions

### 1. 项目结构
采用标准的 Vue3 + Vite 项目结构：
```
src/
├── components/        # Vue 组件
├── composables/       # 组合式函数（TresJS 场景管理、动画控制）
├── assets/
│   └── json/          # 地铁数据 city.json
└── App.vue
```

**替代方案**: 直接在组件内写 Three.js 代码 → **选择**: 抽取到 composables 便于复用和维护

### 2. TresJS 场景组织
- 使用 `<OrbitControls />` (from @tresjs/cientos) 实现视角控制
- 线路渲染使用 `<TresMesh>` + `<TresTubeGeometry>` 或 `<TresLine2>` (from @tresjs/cientos)
- 站点使用 `<TresMesh>` + `<TresSphereGeometry>` 或 `<TresCylinderGeometry>`
- 站点名称使用 `<TresText>` (from @tresjs/cientos) 或 Sprite + CanvasTexture
- 站点上方放置旋转箭头动画
- 列车使用 `<TresMesh>` + `<TresBoxGeometry>`（简化为小立方体）

### 3. 后期处理效果 (@tresjs/post-processing)
- **UnrealBloom**: 霓虹辉光效果，让地铁线路发光
- **Glitch**: 可选的故障艺术效果（用于特色展示）
- **SMAA**: 抗锯齿，提升画质
- **HolographicMaterial**: 全息材质，配合 bloom 效果营造赛博朋克风格

### 4. 动画实现
- 使用 TresJS 的响应式系统驱动动画
- 列车位置沿 `CatmullRomCurve3` 曲线的 `t` 值（0-1）插值
- 使用 Vue `watch` 或 `requestAnimationFrame` 更新列车位置
- 每个列车存储当前的 `t` 值，速度不同的列车 `t` 增量不同

### 4.1 地铁线路呼吸效果
**效果描述**：地铁线路像有生命一样周期性"呼吸"，呈现亮度/透明度的节奏性变化，营造能量流动和赛博朋克氛围。

**技术实现**：
- 使用 `THREE.TubeGeometry` + 自定义 ShaderMaterial 或 `MeshBasicMaterial` + 透明度动画
- 呼吸周期：3-4 秒一个周期（sin 曲线）
- 呼吸参数：
  - 亮度变化：0.6 → 1.0 → 0.6（基础亮度到峰值亮度）
  - 透明度变化：0.7 → 1.0 → 0.7
  - 可选：沿线路方向的波动效果（能量从起点流向终点）

**视觉参数**：
- 呼吸周期：3500ms
- 亮度范围：0.6 - 1.0
- 相位偏移：各线路错开呼吸，避免所有线路同时"吸气/呼气"

```
亮度
 1.0 ─────●─────●─────●─────● 最大亮度
         / \   / \   / \
 0.6 ─●───   ───   ───   ─── 最小亮度
      │←──────── 3.5s ────────→│
```

**与 UnrealBloom 协同**：
- 呼吸时 Bloom 强度同步变化，峰值时线路发光更强烈
- 形成"脉动"的生命感

### 5. 列车粒子组装动画
**效果描述**：页面加载时，列车初始位置上方是一团随机扩散的粒子（类似能量球），2秒后粒子快速砸向初始站位置，凝聚成一辆具有粒子感和赛博朋克风格的小车，然后开始沿线路运行。

**技术实现**：
- 使用 `THREE.Points` 或 `@tresjs/cientos` 的粒子系统
- 粒子初始状态：位置在初始站上方 5-10 单位范围内随机分布，带有微弱上浮动画
- 粒子属性：霓虹色（线路颜色）、发光效果、轻微透明度
- 凝聚动画（2s）：
  - Phase 1 (0-0.5s): 粒子开始收缩，向小车形状的终点位置聚拢
  - Phase 2 (0.5-1.5s): 粒子加速砸向目标，形成拖尾效果（motion blur 粒子轨迹）
  - Phase 3 (1.5-2s): 粒子完全凝聚成小车形状，颜色亮度骤增（能量释放）
- 凝聚完成后：粒子形态转变为实体小车，开始沿 `CatmullRomCurve3` 运行
- 运行时：保留少量残余粒子环绕小车，营造能量尾迹效果

**视觉参数**：
- 粒子数量：200-500 个
- 粒子大小：0.05-0.15 单位
- 凝聚时间：2000ms（ease-in 曲线，前慢后快）
- 小车尺寸：长 1.5 x 宽 0.8 x 高 0.6 单位

### 6. 数据格式
`city.json` 原始数据结构：
```json
{
  "s": "杭州市地铁",
  "i": "3301",
  "l": [{
    "ln": "1号线",
    "cl": "FC0601",
    "st": [{
      "n": "湘湖",
      "en": "Xianghu",
      "p": "1662 1158",
      "sl": "120.234391,30.167585",
      "lg": "2"
    }]
  }]
}
```

数据处理后用于 Three.js：
- 坐标使用网格坐标 `p`（如 "1662 1158"）直接作为 x, z（y 设为 0）
- 线路颜色使用 `cl` 字段

## Risks / Trade-offs

- [风险]: 大规模线路渲染性能 → **缓解**: 使用 `v-for` 渲染优化，或按需渲染可见区域
- [风险]: 曲线路径插值不平滑 → **缓解**: 使用 CatmullRomCurve3 并调整点数
- [风险]: 后期处理影响性能 → **缓解**: 只在高端设备启用 bloom，合理设置阈值
- [风险]: TresJS 响应式动画与 Three.js 原生动画混用 → **缓解**: 统一使用 Vue 响应式驱动
- [风险]: 粒子组装动画计算量大 → **缓解**: 限制粒子数量在 500 以内，使用 Points 而非独立 Mesh
- [风险]: 呼吸效果 + Bloom 同时计算量大 → **缓解**: 线路数量控制在 2 条，bloom 分辨率适当降低
